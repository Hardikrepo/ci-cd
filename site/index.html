Three jobs, one push

Every commit to main runs the same three-stage workflow.

  1. Build

    Lint every HTML file with html-validate, then assemble the output into a versioned artifact.

  2. →

  3. Test

    Crawl the built artifact with linkinator and fail the run on any broken internal link.

  4. →

  5. Deploy

    Assume an AWS role via OIDC, sync to S3, and invalidate the CloudFront cache. Main branch only.

What this setup gets you

No stored AWS keys

GitHub's OIDC token is exchanged for short-lived credentials at deploy time. Nothing long-lived sits in repo secrets.

Locked-down bucket

Public access is fully blocked on the S3 bucket. Only CloudFront, via Origin Access Control, can read objects.

Repo + environment scoped

The IAM trust policy checks the OIDC subject against this exact repository and the production environment.

Automatic cache busting

Every deploy fires a CloudFront invalidation, so visitors never see a stale build after a push.

Fails before it ships

Broken HTML or a dead link stops the pipeline at the test stage — before anything reaches production.

Fully reproducible

The bucket, distribution, and IAM role are all defined in Terraform — apply rebuilds the whole stack.
